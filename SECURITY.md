# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in **pyfake**, please report it responsibly.

* Do **not** open a public GitHub issue
* Email: [praneshmukherjee7@gmail.com](mailto:praneshmukherjee7@gmail.com)

Please include:

* A clear description of the issue
* Steps to reproduce (if possible)
* Potential impact

## What to expect

* You will receive an acknowledgment within a few days
* If the issue is confirmed, we will work on a fix and release it as soon as possible
* Credit may be given for responsible disclosure

## Scope

Since `pyfake` is a fake data generation library, most issues will be treated as bugs rather than security vulnerabilities.
However, anything that could lead to unintended code execution or misuse will be handled as a security issue.
